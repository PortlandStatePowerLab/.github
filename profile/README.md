## 5. Repository Naming Conventions
To keep our organization clean and searchable, all new repositories must follow the snake case naming convention. For example:

`[developer's name]-[funding organization]-[project name]-[starting year]`

Here is an example:

* **developer's name:** `john`
* **funding organization:** `neea`, `pge`, `doe`
* **project name:** `flex-load-calculator`, `wh-conformance-testing`, `egot-derms`
* **starting year:** `26`
* *Example:* `john-neea-egot-derms-26`.

## 6. Creating a New Repository
Do not start a repository from scratch. When spinning up a new project:
1. Click the **New** repository button in the organization.
2. Under the **Repository template** dropdown, select our official README organization template.
3. This ensures your project instantly inherits the standard layout, a predefined README template.
